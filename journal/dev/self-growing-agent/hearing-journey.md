# 我的 AI Agent 终于能听懂我说话了——从"听不见"到"实时对话"的 20 次迭代

> 这是「自生长 Agent 开发日志」系列的第一篇文章。我会记录这个 AI 系统从零到一的完整生长过程，包括每个功能背后的思考、踩坑和妥协。如果你对 AI Agent 开发感兴趣，欢迎关注。

## 从一个荒谬的想法开始

两天前，我产生了一个想法：**给一个 AI Agent 一个角色和目标，让它自己长成什么样就是什么样。**

不是那种你写好所有 prompt、安排好所有流程的 Agent。而是一个真正"活着"的系统——它有自己的记忆、自己的好奇心、自己的思考节奏。就像养育一个孩子，你不能规定它长成什么样，但你可以给它一个环境，让它自己探索。

项目叫 **self-growing-agent**。

第一天，它学会了心跳（每隔几秒自动思考一次）、记忆（工作记忆 + 长期记忆）、工具调用（能读写文件）。到晚上，它已经能自己探索项目结构、搜索记忆、执行命令了。

一切都很顺利。直到我想做一件事：**让它能听见我说话。**

## 为什么"听见"这么难？

你可能觉得，让 AI 听话有什么难的？调个语音 API 不就行了？

但"听见"和"转文字"是两回事。真正的听觉系统需要解决几个问题：

1. **什么时候在听？** —— 你不能 7x24 小时开着麦克风，需要知道什么时候有人在说话
2. **什么时候说完？** —— 需要检测语音活动的开始和结束
3. **噪音怎么办？** —— 键盘声、空调声、你自己清嗓子的声音
4. **怎么响应？** —— 听到之后，是立刻回答还是等你确认？
5. **进程管理** —— 麦克风是独占资源，多个进程抢会出问题

这比看起来复杂得多。

## 第一版：能用，但很蠢

最初的方案很简单：开一个独立进程，定时录音，转成文字，写到文件里。主进程用 `fs.watch` 监听文件变化，读到新内容就触发思考。

```
麦克风 → 录音 → 转文字 → 写文件 → 主进程读取 → 思考 → 回复
```

能跑，但问题一堆：

- 独立进程和主进程抢资源，互相卡死
- 没有语音活动检测（VAD），不管有没有人说话都在录
- 录完一段才处理，延迟好几秒
- `npm start` 要手动启动两个进程

## v2 架构：推倒重来

在修修补补了几个小时后，我做了一个决定：**推倒重来，用事件驱动架构重写。**

新的设计是这样的：

```
感知层（麦克风 → VAD → STT）
    ↓ 事件
思考层（LLM 推理）
    ↓ 事件
行动层（工具调用 → 语音输出）
    ↓ 事件
记忆层（存储 + 检索）
```

所有层通过事件总线通信。每一层只关心自己的事，不需要知道其他层的实现细节。这带来的好处是巨大的——我可以在不改任何其他代码的情况下，单独升级听觉层。

## 真正的故事：20 次迭代

架构搭好后，真正的战斗开始了。以下是接下来几个小时发生的事情：

### 09:17 — 听觉层 v2 集成

第一次把真实麦克风录音接入事件驱动架构。能录音了，能转文字了，但还不能连续对话。

### 09:23 — 语音输出

加上了 TTS 语音输出。Agent 不仅能"听"，还能"说"。但发现了消息重复的 bug——同一个问题被回答了两次。

### 09:36 — 实时听觉 v3

引入了语音活动检测（VAD）。现在 Agent 能自己判断"这个人说完了"，然后开始思考和回答。**这是第一个可以真正对话的版本。**

### 09:45 — 5 个问题修复

看起来能用了，但实际测试发现 5 个问题同时爆发。一个一个修。

### 10:00 — 实时语音对话功能完成

标志性的 commit。这时候，我对着麦克风说话，Agent 能实时听到、思考、然后用语音回答。延迟在 2 秒左右。

但故事还没结束。

### 10:10 — 听觉内嵌主进程

把听觉进程从独立进程改为主进程的子模块。现在 `npm start` 一键启动，不用手动管理两个进程了。

### 10:18 — 非阻塞心跳

发现了一个严重问题：心跳和思考层是阻塞的，导致语音对话时卡顿。改成非阻塞后，对话终于流畅了。

### 10:26-10:57 — 静音阈值的噩梦

这是最折磨人的部分。**静音阈值**决定了"什么算安静、什么算有人在说话"。

- 先从 -50dB 调到 -45dB，想减少噪音误触发
- 调到 -40dB，结果太严格了，人说话都录不到
- 又调回 -50dB

来回折腾了 4 次，每次都要改参数、重启、对着麦克风测试。

同时还在修 STT 解析的问题——Python 的 JSON 提取逻辑被误删了，导致语音转文字的结果无法被正确解析。

## 为什么这段经历值得分享

你可能觉得"修 bug"不值得写。但正是这些 bug 揭示了一个真相：

**AI Agent 开发中，80% 的时间不是在写新功能，而是在调参数、修边界情况、处理进程间通信。**

每一个 commit message 背后都是一次"试了不行、改、再试"的循环。-50dB 和 -40dB 之间差了 10dB，但这 10dB 决定了你的 Agent 是"能用"还是"好用"。

## 现在的它长什么样

经过这 20 次迭代，self-growing-agent 现在具备了：

- **实时语音对话**：对着麦克风说话，2 秒内得到语音回复
- **VAD 语音检测**：自动判断说话开始和结束
- **事件驱动架构**：所有模块通过事件总线解耦
- **一键启动**：`npm start` 启动整个系统
- **7 个版本的听觉系统**：从 v1 到 v7，每个版本解决不同的问题
