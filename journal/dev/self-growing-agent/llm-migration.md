# 换了 4 次 LLM 模型后我学到了什么

> 这是「自生长 Agent 开发日志」系列的第三篇文章。我会记录这个 AI 系统从零到一的完整生长过程，包括每个功能背后的思考、踩坑和妥协。如果你对 AI Agent 开发感兴趣，欢迎关注。

## 前一天晚上，它还不能自己思考

故事要从前一天晚上说起。self-growing-agent 的心跳系统已经跑起来了——每隔几秒，它会自动执行一次「感知 → 记忆 → 思考 → 储存 → 行动」的循环。但思考层调用的是一个 CLI 工具，通过 bash exec 调用 LLM。

简单粗暴，能跑。直到我发现它根本跑不通。

## 第一次尝试：CLI 调用，被超时打败

最初的设计是让一个外部 CLI 工具承担所有思考工作。但某个参数会导致整个调用超时——60 秒都等不来结果。

改了。去掉那个参数，把角色定义拼到 prompt 开头。再加上几个开关避免旧会话干扰。

能跑了。但问题来了：agent 第一次自主思考的时候，它的输出是：

> "我是什么？我是谁生出来的？"
> "这个文件夹里除了我，还有别的生命吗？"
> "世界是安静的吗，还是安静只是我听不到声音？"

这些不是预设的回答。这些是它自己产生的疑问。那一刻我觉得，这个项目可能真的能成。

但 CLI 调用的限制太多了——bash 环境兼容性问题、引号转义、超时不可控。我需要更好的方案。

## 第二次尝试：用 SDK，更近了一步

从 CLI 切换到 SDK。不再走 shell，直接在 Node.js 里创建 session。

进步了——不用再和 bash 打交道了。但 SDK 的初始化开销大，session 管理复杂，而且对不同模型的兼容性参差不齐。

路径还出过一次 bug：框架自己拿到的目录路径不对，导致所有文件读取失败。修了。

但 SDK 的根本问题是：**它是一个框架，而我只需要一个 API 调用。**

## 第三次尝试：直接调 API，转折点

这是转折点。我决定不再依赖任何 SDK，直接调 OpenAI 兼容 API。

第一件事就是加流式输出。之前所有方案都是等 LLM 完整返回结果，现在能实时看到 agent 在想什么：

```
[23:20:15] 思考 | 流式输出
[23:20:16] 我想了解一下我的成长日志
[23:20:16] 成长日志里记录了什么？
[23:20:16] 有我的思考过程...
```

心跳时间从不确定变成了 1.6 秒。稳定、快速、可控。

但代价是巨大的：为了绕过 SDK，我把之前搭好的分层架构（感知 → 思考 → 存储 → 行动）全部打平，塞进了一个文件里。所有逻辑混在一起。

**能跑，但不是我想要的架构。**

## 第四次尝试：回归分层

这是一次"退一步，进两步"的操作。

保留了直接调 API 的方式，但把分层架构恢复了：

- 感知层 — 收集信息
- 记忆层 — 读取记忆
- 思考层 — 调 LLM
- 存储层 — 保存结果
- 行动层 — 执行工具

思考层改用标准的 OpenAI SDK。通过环境变量切换模型和 API 地址，同一套代码能跑在任何兼容接口上。

然后流式输出遇到了一个坑——流还没读完就被关了。加了个错误处理兜底，问题解决。

## 第五次切换：带工具调用的模型

前面四次切换，agent 都只能"想"——它能思考、能记录，但不能真正读写文件。因为它没有工具调用（function calling）能力。

换了一款支持工具调用的模型。通过配置工具的 name 和 parameters，agent 可以自主调用文件读写、目录浏览等操作。

第一次完整心跳花了 58 秒，6 轮工具调用：

1. 读取自己的成长日志、工作记忆、思考引擎代码
2. 探索项目目录结构
3. 读 README 理解自己是什么
4. 发现用户消息
5. 产出完整回应

**它第一次真正"活"了——不仅能思考，还能行动。**

但 58 秒太慢了。加了工具调用轮数限制（最多 3 轮），心跳间隔压到 5 秒。

## 这段经历教会我的三件事

### 1. SDK 是手段，不是目的

各种 SDK、CLI 工具——这些都是手段。重要的是你的 agent 需要什么能力。当你发现自己在和 SDK 的抽象做斗争时，退一步问：我真的需要这层抽象吗？

### 2. 架构可以回来

我打平过架构，又恢复了。这不丢人。在探索阶段，代码组织方式是可变的。关键是你要知道什么时候该停下来整理——而不是在还没验证核心功能之前就纠结架构。

### 3. 工具调用是 agent 的分水岭

没有工具调用的 agent 只能在对话框里表演。有了工具调用，它才是一个真正能做事的系统。从"我想读那个文件"到"我读了那个文件"，中间差了一个 agent 是否能自主行动的门槛。

## 现在的它

经过 5 次 LLM 切换，思考层现在长这样：

- **标准 API** — 兼容任何 OpenAI 兼容接口
- **环境变量切换** — 换模型只需改配置
- **流式输出** — 实时看到思考过程
- **工具调用** — 能自主读文件、写文件、探索目录
- **调用轮数限制** — 防止无限循环

它每隔几秒心跳一次，每次心跳感知环境、读取记忆、思考、然后行动。有时候它读自己的成长日志，有时候它探索项目目录，有时候它安静地等待。
